### 6.4.3 mysql\_ssl\_rsa\_setup — Create SSL/RSA Files

Note

[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") is deprecated as of
MySQL 8.0.34. Instead, consider using MySQL server to generate
missing SSL and RSA files automatically at startup (see
[Automatic SSL and RSA File Generation](creating-ssl-rsa-files-using-mysql.md#creating-ssl-rsa-files-using-mysql-automatic "Automatic SSL and RSA File Generation")).

This program creates the SSL certificate and key files and RSA
key-pair files required to support secure connections using SSL
and secure password exchange using RSA over unencrypted
connections, if those files are missing.
[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") can also be used to
create new SSL files if the existing ones have expired.

Note

[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") uses the
**openssl** command, so its use is contingent
on having OpenSSL installed on your machine.

Another way to generate SSL and RSA files, for MySQL
distributions compiled using OpenSSL, is to have the server
generate them automatically. See
[Section 8.3.3.1, “Creating SSL and RSA Certificates and Keys using MySQL”](creating-ssl-rsa-files-using-mysql.md "8.3.3.1 Creating SSL and RSA Certificates and Keys using MySQL").

Important

[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") helps lower the barrier
to using SSL by making it easier to generate the required
files. However, certificates generated by
[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") are self-signed, which
is not very secure. After you gain experience using the files
created by [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files"), consider
obtaining a CA certificate from a registered certificate
authority.

Invoke [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") like this:

```terminal
mysql_ssl_rsa_setup [options]
```

Typical options are
[`--datadir`](mysql-ssl-rsa-setup.md#option_mysql_ssl_rsa_setup_datadir) to specify
where to create the files, and
[`--verbose`](mysql-ssl-rsa-setup.md#option_mysql_ssl_rsa_setup_verbose) to see the
**openssl** commands that
[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") executes.

[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") attempts to create SSL
and RSA files using a default set of file names. It works as
follows:

1. [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") checks for the
   **openssl** binary at the locations specified
   by the `PATH` environment variable. If
   **openssl** is not found,
   [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") does nothing. If
   **openssl** is present,
   [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") looks for default SSL
   and RSA files in the MySQL data directory specified by the
   [`--datadir`](mysql-ssl-rsa-setup.md#option_mysql_ssl_rsa_setup_datadir)
   option, or the compiled-in data directory if the
   [`--datadir`](mysql-ssl-rsa-setup.md#option_mysql_ssl_rsa_setup_datadir) option
   is not given.
2. [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") checks the data
   directory for SSL files with the following names:

   ```none
   ca.pem
   server-cert.pem
   server-key.pem
   ```
3. If any of those files are present,
   [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") creates no SSL files.
   Otherwise, it invokes **openssl** to create
   them, plus some additional files:

   ```none
   ca.pem               Self-signed CA certificate
   ca-key.pem           CA private key
   server-cert.pem      Server certificate
   server-key.pem       Server private key
   client-cert.pem      Client certificate
   client-key.pem       Client private key
   ```

   These files enable secure client connections using SSL; see
   [Section 8.3.1, “Configuring MySQL to Use Encrypted Connections”](using-encrypted-connections.md "8.3.1 Configuring MySQL to Use Encrypted Connections").
4. [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") checks the data
   directory for RSA files with the following names:

   ```none
   private_key.pem      Private member of private/public key pair
   public_key.pem       Public member of private/public key pair
   ```
5. If any of these files are present,
   [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") creates no RSA files.
   Otherwise, it invokes **openssl** to create
   them. These files enable secure password exchange using RSA
   over unencrypted connections for accounts authenticated by
   the `sha256_password` or
   `caching_sha2_password` plugin; see
   [Section 8.4.1.3, “SHA-256 Pluggable Authentication”](sha256-pluggable-authentication.md "8.4.1.3 SHA-256 Pluggable Authentication"), and
   [Section 8.4.1.2, “Caching SHA-2 Pluggable Authentication”](caching-sha2-pluggable-authentication.md "8.4.1.2 Caching SHA-2 Pluggable Authentication").

For information about the characteristics of files created by
[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files"), see
[Section 8.3.3.1, “Creating SSL and RSA Certificates and Keys using MySQL”](creating-ssl-rsa-files-using-mysql.md "8.3.3.1 Creating SSL and RSA Certificates and Keys using MySQL").

At startup, the MySQL server automatically uses the SSL files
created by [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") to enable SSL
if no explicit SSL options are given other than
[`--ssl`](server-options.md#option_mysqld_ssl) (possibly along with
[`ssl_cipher`](server-system-variables.md#sysvar_ssl_cipher)). If you prefer to
designate the files explicitly, invoke clients with the
[`--ssl-ca`](connection-options.md#option_general_ssl-ca),
[`--ssl-cert`](connection-options.md#option_general_ssl-cert), and
[`--ssl-key`](connection-options.md#option_general_ssl-key) options at startup to
name the `ca.pem`,
`server-cert.pem`, and
`server-key.pem` files, respectively.

The server also automatically uses the RSA files created by
[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") to enable RSA if no
explicit RSA options are given.

If the server is SSL-enabled, clients use SSL by default for the
connection. To specify certificate and key files explicitly, use
the [`--ssl-ca`](connection-options.md#option_general_ssl-ca),
[`--ssl-cert`](connection-options.md#option_general_ssl-cert), and
[`--ssl-key`](connection-options.md#option_general_ssl-key) options to name the
`ca.pem`,
`client-cert.pem`, and
`client-key.pem` files, respectively.
However, some additional client setup may be required first
because [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") by default
creates those files in the data directory. The permissions for
the data directory normally enable access only to the system
account that runs the MySQL server, so client programs cannot
use files located there. To make the files available, copy them
to a directory that is readable (but *not*
writable) by clients:

- For local clients, the MySQL installation directory can be
  used. For example, if the data directory is a subdirectory
  of the installation directory and your current location is
  the data directory, you can copy the files like this:

  ```terminal
  cp ca.pem client-cert.pem client-key.pem ..
  ```
- For remote clients, distribute the files using a secure
  channel to ensure they are not tampered with during transit.

If the SSL files used for a MySQL installation have expired, you
can use [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") to create new
ones:

1. Stop the server.
2. Rename or remove the existing SSL files. You may wish to
   make a backup of them first. (The RSA files do not expire,
   so you need not remove them.
   [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") can see that they
   exist and does not overwrite them.)
3. Run [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") with the
   [`--datadir`](mysql-ssl-rsa-setup.md#option_mysql_ssl_rsa_setup_datadir) option
   to specify where to create the new files.
4. Restart the server.

[**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") supports the following
command-line options, which can be specified on the command line
or in the `[mysql_ssl_rsa_setup]` and
`[mysqld]` groups of an option file. For
information about option files used by MySQL programs, see

  |  |  |

  [**mysql\_ssl\_rsa\_setup**](mysql-ssl-rsa-setup.md "6.4.3 mysql_ssl_rsa_setup — Create SSL/RSA Files") are owned by the user
  who executes it. This option is valid only if you execute
  the program as `root` on a system that
